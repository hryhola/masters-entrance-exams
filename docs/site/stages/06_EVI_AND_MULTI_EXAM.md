# Етап 6. ЄВІ та кілька форматів іспиту

## Ціль

Довести, що архітектура підтримує не лише single-choice ЄФВВ, та
підключити перші підготовлені набори ЄВІ.

## Роботи

- проаналізувати й нормалізувати джерела ЄВІ;
- розширити `ExamDefinition` правилами конкретного іспиту;
- підтримати секції одного іспиту;
- додати потрібні типи відповідей;
- реалізувати текстові контексти, спільні для кількох питань;
- додати мовні налаштування й специфічну типографіку;
- реалізувати окремі правила таймера та оцінювання;
- об'єднати статистику на рівні іспиту без змішування різних шкал;
- додати міграцію `localStorage` для нових типів прогресу.

## Можливі типи завдань

- одна правильна відповідь;
- кілька правильних відповідей;
- встановлення відповідностей;
- заповнення пропуску;
- коротка текстова відповідь;
- група питань до одного тексту.

Реалізовуються лише типи, які підтверджені фактичною структурою
офіційних матеріалів ЄВІ.

## Результат

Каталог містить ЄФВВ та ЄВІ, а спільний тестовий рушій проходить обидва
іспити через конфігурацію та спеціалізовані компоненти типів відповіді.

## Критерії приймання

- ЄФВВ продовжує працювати без зміни його dataset;
- кожен новий тип питання має keyboard-friendly UI;
- правила оцінювання тестуються окремо;
- прогрес різних іспитів не конфліктує;
- видалення одного dataset не пошкоджує історію інших.
