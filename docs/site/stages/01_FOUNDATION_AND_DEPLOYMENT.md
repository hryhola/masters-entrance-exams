# Етап 1. Каркас застосунку та деплой

## Ціль

Створити мінімальний React-застосунок, який локально запускається,
проходить перевірки та автоматично публікується на GitHub Pages.

## Роботи

- створити `app/` на React, TypeScript і Vite;
- налаштувати форматування, lint, перевірку типів і тести;
- додати hash router і базову карту сторінок;
- створити адаптивний application shell: header, навігація, main;
- визначити дизайн-токени для кольорів, відступів і типографіки;
- додати сторінки-заглушки для каталогу, практики й прогресу;
- налаштувати Vite `base` для локального та GitHub Pages оточення;
- створити GitHub Actions workflow для перевірки та деплою;
- додати сторінки помилки й недоступного маршруту.

## Результат

За кожного push у головну гілку перевірений статичний сайт автоматично
публікується на GitHub Pages.

## Критерії приймання

- `npm run dev` запускає застосунок;
- `npm run build` завершується успішно;
- lint, typecheck і unit-тести мають окремі команди;
- пряме оновлення будь-якого hash-маршруту працює;
- production-збірка відкривається з repository base path;
- навігація доступна з клавіатури;
- GitHub Pages workflow не публікує сайт після невдалих тестів.

## Не входить

- реальні питання;
- збереження прогресу;
- тестова логіка;
- фінальний візуальний дизайн.
