# Етап 7. Згенерований контент і стабільний реліз

## Ціль

Додати контрольоване джерело нових завдань і підготувати сайт до
регулярного використання без змішування офіційного та згенерованого
контенту.

## Роботи з контентом

- визначити окремий pipeline генерації;
- генерувати за іспитом, темою, складністю й типом;
- перевіряти схему, відповідь і пояснення;
- додати обов'язковий редакторський статус;
- публікувати тільки затверджені набори;
- маркувати `origin: generated`;
- зберігати модель, prompt, дату й версію генератора;
- виключати близьке копіювання офіційних формулювань.

## Роботи зі стабілізації

- додати наскрізні browser-тести;
- перевірити accessibility;
- оптимізувати завантаження наборів і зображень;
- додати сторінку про джерела й обмеження;
- перевірити мобільні браузери;
- додати резервне відновлення прогресу;
- документувати release checklist;
- зафіксувати політику оновлення dataset і міграцій.

## Результат

Користувач може продовжувати практику після вивчення офіційних тестів,
чітко бачачи походження й рівень перевірки кожного нового питання.

## Критерії приймання

- неперевірений згенерований набір не потрапляє в production;
- офіційні та згенеровані питання мають різне видиме маркування;
- статистику можна фільтрувати за походженням;
- production build проходить unit, integration і browser-тести;
- повторний GitHub Pages deploy не пошкоджує локальний прогрес;
- основні сценарії відповідають узгодженим вимогам доступності.
