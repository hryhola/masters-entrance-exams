# Вимоги до навчального сайту

## Користувацька ціль

Користувач має розуміти, що вчити сьогодні, мати змогу пройти тренування
та отримати зрозумілий результат: правильні відповіді, пояснення,
слабкі теми й наступні кроки.

## Основні сценарії

### Перше відвідування

1. Користувач бачить доступні іспити та стан їхнього контенту.
2. Обирає ЄФВВ з інформаційних технологій.
3. Запускає повний тест, тему або коротку сесію.
4. Після першої завершеної сесії сайт створює локальний профіль прогресу.

### Щоденне тренування

1. Головна сторінка показує рекомендовану сесію.
2. До неї входять помилки, прострочені повторення та слабкі теми.
3. Після проходження оновлюються рівень засвоєння й дата повторення.

### Симуляція іспиту

1. Користувач запускає повний тест із таймером.
2. Відповіді не перевіряються до завершення.
3. Після завершення показуються бал, час, теми й детальний розбір.

### Навчання на помилках

1. Користувач відкриває список неправильних відповідей.
2. Переглядає пояснення до правильного й обраного варіанта.
3. Повторно відповідає на питання пізніше, а не одразу запам'ятовує
   позицію правильної літери.

## Обов'язкові функції MVP

- каталог іспитів;
- повний тест ЄФВВ 2024;
- тематична та швидка сесії;
- текст, Markdown, формули, код, таблиці й зображення;
- вибір відповіді, навігація та позначка для повернення;
- режим навчання з миттєвим поясненням;
- режим іспиту з відкладеним результатом;
- підсумок сесії та перегляд помилок;
- локальна історія спроб;
- базова статистика за темами;
- відновлення незавершеної сесії після перезавантаження.

## Функції після MVP

- інтервальне повторення;
- щоденна рекомендована сесія;
- закладки;
- імпорт та експорт локального прогресу;
- календар активності;
- тести ЄВІ;
- згенеровані тренувальні питання;
- встановлення цільової дати іспиту;
- оцінка готовності на основі останніх спроб.

## Принципи інтерфейсу

- мобільний екран є повноцінним, а не скороченим режимом;
- під час відповіді увага зосереджена на одному питанні;
- офіційний і згенерований контент завжди візуально розрізняються;
- спірні офіційні ключі мають помітне попередження;
- пояснення не перекриває формулювання питання;
- колір не є єдиним способом позначити правильність;
- таймер не повинен заважати режиму спокійного навчання.

## Початкова карта сторінок

| Маршрут | Призначення |
|---|---|
| `/` | Головна панель і продовження навчання |
| `/exams` | Каталог іспитів |
| `/exams/:examId` | Опис і режими вибраного іспиту |
| `/practice/setup` | Налаштування сесії |
| `/practice/:sessionId` | Проходження питань |
| `/results/:sessionId` | Результат і розбір |
| `/review` | Помилки, повторення й закладки |
| `/progress` | Статистика за темами та періодами |
| `/settings` | Дані, вигляд і параметри навчання |

У фактичній URL-адресі GitHub Pages ці маршрути розташовуватимуться після
символу `#`.

## Критерії успіху продукту

- користувач може пройти весь тест без втрати стану;
- кожне питання коректно відображається на мобільному й широкому екрані;
- результат пояснює не тільки бал, а й слабкі теми;
- повторне тренування формується з реальної історії відповідей;
- додавання нового іспиту не потребує переписування тестового рушія;
- оновлення структури `localStorage` не знищує прогрес без попередження.

## Поза MVP

- реєстрація та авторизація;
- серверна база даних;
- синхронізація між пристроями;
- соціальні рейтинги;
- платні функції;
- редактор контенту безпосередньо в браузері;
- автоматична публікація неперевірених згенерованих питань.
